DIAGRAMA DE APIS E INTERFACES - SISTEMA GUINCHO LACERDA

1. ENDPOINTS DA API

[Autenticação]
  ├── /api/auth/login
  │   ├── POST: Login usuário
  │   └── Response: JWT Token
  │
  ├── /api/auth/refresh
  │   └── POST: Renovar token
  │
  └── /api/auth/logout
      └── POST: Encerrar sessão

[Usuários]
  ├── /api/usuarios
  │   ├── GET: Listar usuários
  │   └── POST: Criar usuário
  │
  └── /api/usuarios/{id}
      ├── GET: Detalhes usuário
      ├── PUT: Atualizar usuário
      └── DELETE: Desativar usuário

[Serviços]
  ├── /api/servicos
  │   ├── GET: Listar serviços
  │   └── POST: Criar serviço
  │
  ├── /api/servicos/{id}
  │   ├── GET: Detalhes serviço
  │   ├── PUT: Atualizar serviço
  │   └── DELETE: Cancelar serviço
  │
  └── /api/servicos/{id}/anexos
      ├── POST: Adicionar anexo
      └── GET: Listar anexos

[Transações]
  ├── /api/transacoes
  │   ├── GET: Listar transações
  │   └── POST: Criar transação
  │
  └── /api/transacoes/{id}
      ├── GET: Detalhes transação
      ├── PUT: Atualizar transação
      └── POST: Aprovar/Rejeitar

5. RESPONSES PADRÃO

[Sucesso]
  ├── 200: OK
  │   └── {data, message}
  │
  ├── 201: Created
  │   └── {id, message}
  │
  └── 204: No Content

[Erro]
  ├── 400: Bad Request
  │   └── {error, details}
  │
  ├── 401: Unauthorized
  │   └── {error, redirect}
  │
  └── 500: Server Error
      └── {error, trace}
