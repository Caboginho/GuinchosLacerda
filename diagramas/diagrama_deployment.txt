DIAGRAMA DE DEPLOYMENT - SISTEMA GUINCHO LACERDA

1. AMBIENTE DE PRODUÇÃO

[Cliente Web] <<device>>
  |
  ├── [Navegador] <<runtime>>
  |     ├── Interface HTML/CSS/JS
  |     ├── Bootstrap Components
  |     └── AJAX Client
  |
  └── [Cache Local] <<storage>>
        └── LocalStorage/SessionStorage

[Servidor Aplicação] <<device>>
  |
  ├── [Flask Server] <<runtime>>
  |     ├── Python Environment
  |     ├── Blueprint Modules
  |     └── Session Manager
  |
  ├── [SQLite] <<database>>
  |     ├── Data Files
  |     └── Backup Files
  |
  └── [File System] <<storage>>
        ├── Temp Files
        ├── Logs
        └── Cached Files

[Google Cloud] <<platform>>
  |
  ├── [Drive Service] <<service>>
  |     ├── File Storage
  |     └── Access Control
  |
  └── [Sheets Service] <<service>>
        ├── Data Tables
        └── Formulas

2. CONEXÕES FÍSICAS

[Cliente] <<>> [Servidor]
  ├── Protocol: HTTPS
  ├── Port: 443
  └── Mode: Bi-directional

[Servidor] <<>> [Google Cloud]
  ├── Protocol: OAuth 2.0
  ├── APIs: REST
  └── Security: SSL/TLS

[Server Components]
